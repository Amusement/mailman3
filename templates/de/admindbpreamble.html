Diese Seite enthält die aktuellen administrativen Anfragen für die Mailingliste %(listname)s die Ihre Aufmerksamkeit benötigen. Zuerst sehen Sie die Liste der anstehenden Abonnementsanfragen (falls es welche gibt), gefolgt von Emails, die Ihre Genehmigung erfordern.

Wählen Sie für jede Abonnementsanfragen Refuse oder Subscribe. Sollten Sie eine Abonnementsanfragen abweisen, können Sie optional einen Grund angeben.

Wählen Sie für jede zurückgehalte Email an die Liste eine der Aktionen aus:

  • Defer - Die Entscheidung auf später verschieben. Benutzen Sie dies, wenn Sie die Nachricht weiterleiten oder aufbewahren möchten (siehe unten), Sie aber nicht sofort verwerfen wollen.
  • Approve - Genehmigen und an die Liste senden.
  • Reject - Abweisen und eine Abweisungsnotiz an den Absender schicken. Die ursprüngliche Nachricht wird verworfen.
  • Discard - Die ursprüngliche Nachricht wird verworfen, keine Abweisungsnotiz wird an den Absender geschickt. Dise ist für Spam nützlich.

Aktivieren sie die Preserve Option, wenn Sie eine Kopie der Nachricht für den Site Administrator aufbewahren möchten. Dies ist insbesondere nützlich für unflätige oder beleidigende Nachrichten. Aktivieren Sie die Forward to Option und geben SIe die Weiterleitungsadresse ein, falls Sie die Email an jemanden weiterleiten möchten, der nicht auf der Liste ist.

Wenn Sie fertig sind, klicken sie auf den Submit All Data Knopf am Kopf oder Fuß dieser Seite. Dieser Knopf läßt die von Ihnen getroffene Entscheidungen aktiv werden. Wenn Sie für einige Nachrichten keine Entscheidung treffen, werden diese Nachrichten nicht aus der Datenbank der zu bearbeitenden Anfragen gelöscht.
